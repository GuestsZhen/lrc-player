# 音频调性检测功能说明

## 概述

本项目实现了一个轻量级的浏览器端音频调性检测功能，使用基于色度特征（Chroma Features）和 Krumhansl-Schmiedler 键配置文件的算法。

## 技术方案

### 核心算法

1. **色度特征提取**：将音频信号转换为 12 个半音的能量分布
2. **Krumhansl-Schmiedler 匹配**：使用经典的键配置文件进行相关性匹配
3. **自相关音高估计**：使用时域自相关方法估计基频

### 优势

- ✅ **纯前端实现**：无需后端服务器或在线 API
- ✅ **无需上传文件**：所有处理在浏览器本地完成
- ✅ **轻量级**：不依赖大型 WASM 库
- ✅ **隐私保护**：音频数据不会离开用户设备
- ✅ **免费使用**：无 API 调用限制

### 局限性

- ⚠️ 精度不如专业的音频分析软件
- ⚠️ 对复杂音乐（多乐器、和声丰富）可能不够准确
- ⚠️ 需要播放音频才能分析（无法从 URL 直接分析）

## 使用方法

### 自动检测

当你在 Player 页面播放歌曲时，系统会自动开始调性检测：

1. 选择一首歌曲播放
2. 等待几秒，右上角的 🎵 按钮会显示检测结果
3. 结果显示为调性徽章（如 "C"、"F#" 等）

### 手动检测

如果需要重新检测：

1. 点击右上角的 🎵 按钮
2. 在弹出的菜单中点击"重新检测"
3. 等待检测完成

## 技术细节

### 文件位置

- 核心算法：`src/utils/simple-key-detector.ts`
- UI 组件：`src/components/header.tsx`（调性检测按钮和菜单）
- 业务逻辑：`src/components/content.tsx`（检测触发和结果处理）

### 性能优化

- 只分析前 30 秒音频
- 限制处理的帧数（最多 100 帧）
- 使用简化的频谱分析方法
- 异步处理，不阻塞 UI

### 支持格式

- MP3
- WAV
- FLAC
- 其他浏览器支持的音频格式

## 故障排除

### 检测不准确

**原因**：简化算法的固有限制

**解决方案**：
- 尝试不同的歌曲（独奏、简单和声的歌曲效果更好）
- 确保音频质量良好
- 考虑使用更专业的工具进行验证

### 检测速度慢

**原因**：音频文件过大或设备性能较低

**解决方案**：
- 系统已自动限制分析时长为 30 秒
- 关闭其他占用资源的程序
- 使用性能更好的设备

### 检测失败

**可能原因**：
- 音频格式不支持
- 文件损坏
- 浏览器兼容性问题

**解决方案**：
- 检查控制台错误信息
- 尝试转换音频格式
- 更新浏览器到最新版本
