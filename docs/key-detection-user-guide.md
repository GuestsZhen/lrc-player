# 调性检测功能使用说明

## 🎵 功能概述

在 Player 播放页面右侧，文字设定按钮下方新增了一个**调性检测按钮**（🎵图标），可以实时分析当前播放歌曲的调性（大调/小调）。

## 📍 位置说明

```
右上角控制按钮区域（从上到下）：
┌─────────────┐
│   Player    │  ← 播放按钮
├─────────────┤
│  Fullscreen │  ← 全屏按钮（仅Player页）
├─────────────┤
│     T       │  ← 文字设定按钮
├─────────────┤
│     🎵      │  ← 调性检测按钮（新功能）
└─────────────┘
```

## 🚀 使用方法

### 方法一：自动检测
1. 在文件列表中选择一首歌曲播放
2. 系统会**自动开始检测**该歌曲的调性
3. 检测完成后，按钮上会显示调性缩写（如 "C"、"Am"）

### 方法二：手动检测
1. 点击调性检测按钮（🎵）
2. 弹出调性检测菜单
3. 点击"开始检测"或"重新检测"按钮
4. 等待检测完成（通常需要 2-5 秒）

## 📊 检测结果展示

检测完成后，菜单中会显示：
- **完整调性名称**：如 "C 大调"、"A 小调"
- **检测强度**：0-1 之间的数值，表示检测的可信度
- **重新检测按钮**：可以重新分析当前歌曲

## 💡 应用场景

### 1. 简谱转调辅助
在 Tune（转调）页面中，可以根据检测结果：
- 自动设置原调
- 快速选择目标调
- 进行准确的简谱转调

### 2. 音乐学习
- 了解歌曲的调性结构
- 学习和弦进行
- 练习即兴演奏

### 3. DJ 混音
- 找到调性匹配的歌曲
- 实现和谐的音乐过渡
- 创建更好的播放列表

## ⚙️ 技术细节

### 分析参数
- **分析时长**：默认分析前 30 秒音频
- **算法**：使用 Essentia.js 的 KeyExtractor
- **精度**：支持 12 个大调和 12 个小调
- **性能**：WebAssembly 加速，浏览器端实时处理

### 支持的调性
- **大调**：C, C#, D, D#, E, F, F#, G, G#, A, A#, B
- **小调**：Cm, C#m, Dm, D#m, Em, Fm, F#m, Gm, G#m, Am, A#m, Bm

## 🔧 故障排除

### 问题1：检测按钮不显示
**原因**：不在 Player 页面  
**解决**：确保当前在播放页面（路由为 `#player`）

### 问题2：检测一直进行中
**原因**：音频文件过大或格式不支持  
**解决**：
- 尝试较小的音频文件
- 确保是常见格式（MP3, WAV, FLAC 等）
- 检查浏览器控制台是否有错误

### 问题3：检测结果不准确
**原因**：
- 音频质量较差
- 歌曲调性复杂（转调频繁）
- 分析时长不足

**解决**：
- 使用高质量音频文件
- 尝试重新检测
- 对于复杂歌曲，可能需要人工判断

## 📝 注意事项

1. **首次加载**：第一次使用时，Essentia.js 库需要加载 WebAssembly 文件（约 2-3MB），请耐心等待
2. **内存占用**：分析大型音频文件时会占用较多内存，建议关闭其他标签页
3. **离线使用**：一旦加载完成，可以在离线状态下使用（PWA 支持）
4. **隐私保护**：所有分析都在浏览器本地进行，不会上传到服务器
